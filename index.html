História dos computadores

(Voltar)

Os primeiros computadores foram criados com o propósito de realizar cálculos matemáticos avançados, em menos tempo do que era possível apenas com a ação humana. Principalmente com a necessidade gerada por conta da Guerra.

As gerações dos computadores foram subdivididas em 5 grandes gerações:

  1. 1a geração: com o surgimento do Harvard Mark 1 em 1944 e do Colossus e ENIAC em 1946
  2. 2a geração: de 1959 a 1965, com o avanço da tecnologia para substituir as válulas pelos transistores. O que possibilitou a diminuição do tamanho dos computadores da época
  3. 3a geração: de 1965 a 1970, com o surgimento dos circuitos integrados. Com eles tornou-se possível diminuir ainda mais o tamanho dos computadores.
  4. 4a geração: de 1971 em diante, com o avanço da tecnologia, os circuitos integrados foram substituidos pelos microprocessadores. Tornando esta a era dos "personal computers"
  5. 5a geração: a era da computação quantica, IAs, cripto-moedas e realidade aumentada. Aguns dizem que já estamos nesta era, outros que ainda não

O que são Clients?

(Voltar)

Os clients ou clientes são os dispositivos que irão consumir a informação armazenada nos servidores, através de uma conexão pela internet. Estes dispositivos podem ser desde computadores e celulares, até televisões e sistemas embarcados de carros. Por conta destas diferenças entre dispositivos, é gerada uma preocupação no desenvolvedor que atua nesta área, de levar em consideração a plataforma na qual aquele programa será executado. No caso de aplicações web, a comunicação ocorre da seguinte forma:

  1. Cliente faz uma requisição
  2. Esta requisição trafega via internet
  3. Chega até o servidor que hospeda o site
  4. Este servidor envia os arquivos referentes ao site (HTML, CSS e JavaScript)
  5. Os arquivos fazem o caminho de volta via internet
  6. E são armazenados em cache no dispositivo cliente para utilização

De acordo com este fluxo de comunicação, novos fatores precisam ser levados em consideração. Em aplicações Web, o dispositivo cliente estará fazendo o uso de algum navegador, para interpretar os arquivos do site. Neste ponto, é preciso levar em consideração a compatibilidade do código com os navegadores existentes no mercado. Os navegadores têm uma certa padronização para que seja possível o mesmo site operar da mesma forma na maioria das situações em diferentes navegadores. Porém podem haver exceções.

Outro fator importante é que, por conta da conexão ocorrer via internet entre o cliente e o servidor, tanto a velocidade de internet do cliente, quanto o tamanho dos arquivos do site, precisam ser considerados como um possível problema. Assim, ao desenvolver um site, pensar em uma solução leve em termos de recursos, é sempre uma boa prática.

O que são Servers?

(Voltar)

O servers ou servidores, são dispositivos que irão forncer informações ou serviços ao cliente, por meio da internet ou não (no caso de uma rede local - LAN). Os servidores são máquinas geramente com um hardware robusto e de alto desempenho, a depender de sua atividade. Podem ser desde máquinas únicas como PCs comuns, até Datacenters, que são aglomerados de servidores chegando as vezes a ocupar grandes salas inteiras.

Estas máquinas contam com um sistema operacional específico para servidores, podendo ser de diferentes arquiteturas (Ex.: windows, linux, etc), onde é intalado um servidor web. Portanto no fluxo de comunicação com o cliente, o servidor recebe uma requisição, que é lida pelo sistema operacional e este a direciona para o servidor web, para que seja interpretada e devolvidos os arquivos necessários ao cliente.

Os servidores podem ser de diferentes tipos, que executam tarefas variadas. Alguns tipos de servidores são:

  • Arquivos
  • Segurança
  • Streaming de dados
  • E-mail
  • Web
  • Banco de dados
  • DNS (Domain Name Service)

Como é uma estrutura básica de um HTML?

(Voltar)

O HTML é uma linguagem de marcação de hipertexto (Hypertext Markup Language), que consiste em um arquivo de texto estruturado por tags (marcações), que são interpretadas pelo navegador e assim apresentadas ao usuário de acordo com sua função pré-definida.

Estrutura básica de uma tag ([opicional])

< nome-tag [atributos] >[ < / nome-tag > ]

Note que toda tag, tem uma abertura, identificada pelos sinais de < >, porém o fechamento da tag, sinais de < / >, é opicional a depender da tag que está sendo utilizada. E dentro da tag podem ser passados atributos, que seriam algo como identificadores que modificam a tag de alguma forma. Seja esta mudança visível ao usuário ou não.

Na estrutura de um arquivo HTML, existem 3 tags primordiais:

  • HTML: identifica que o conteúdo do arquivo em questão será de fato um HTMl
  • head: esta tag identifica o cabeçalho da página. E dentro dela, serão registrados parâmetros de pré-processamento, conexões com outros arquivos (ex.: com arquivos de estilo ou com o javascript da página), etc. Iniciados junto ao carregamento da página, que geralmente não são visíveis ao usuário
  • body: seu conteúdo é a parte do corpo da página, ou seja, a área realmente visível ao usuário. Como por exemplo, os títulos, textos ou imagens.

Entendendo a diferença entre listas ordenadas e não ordenadas

(Voltar)

No HTML existem 2 tipos de listas. Listas ordenadas e listas não ordenadas.

As listas ordenadas, identificadas pela tag < ol > < / ol >, são utilizadas quando a ordem de seus itens é relevante em relação a informação sendo apresentada. Por exemplo em uma ordem de acontecimentos ou um passo-a-passo. Seu resultado é conforme exemplificado abaixo.

  1. Item ordenado
  2. Item ordenado
  3. Item ordenado

As listas não ordenadas, identificadas pela tag < ul > < / ul >, são utilizadas quando a ordem de seus itens não é relevante em relação a informação sendo apresentada, ou seja, um item não tem dependência do item anterior. Por exemplo em uma lista de compras, tecnologias, marcas, etc. Seu resultado é conforme exemplificado abaixo.

  • Item não ordenado
  • Item não ordenado
  • Item não ordenado

As listas também podem ter sub itens, tanto no caso de uma lista ordenada quanto no caso de uma lista não ordenada. E seus sub itens podem ou não ser ordenados. Como no exemplo abaixo.

  1. Item ordenado
  2. Item ordenado
    • Sub item não ordenado
    • Sub item não ordenado
  3. Item ordenado
